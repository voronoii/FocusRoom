## Affected Pages/Routes
- `/` (Entry) — task input, time selection, avatar preview, "입장하기" button
- `/room` (Room) — avatar grid, timer bar, "완료" button, ambient sound toggle
- `/result` (Result) — completion/timeout display, MyPage history, share card, email capture

## Key Interactions to Verify
- 할 일 입력 + 시간 선택 → 입장 → 방 표시 on Entry page
- 타이머 카운트다운이 정확히 동작하는지 on Room page
- "완료" 버튼 클릭 → confetti + 상태 변경 on Room page
- 타이머 만료 시 자동 리다이렉트 on Room page
- "다시 입장" 클릭 → 새 아바타로 Entry on Result page
- 공유 이미지 생성 + 공유 on Result page
- 선택적 이메일 입력 on Result page

## Edge Cases
- 더블 클릭 방지 (입장, 완료 버튼) on Entry/Room
- 오프라인 상태에서 타이머 동작 + 재연결 on Room
- 방 가득 참 (30명) 시 입장 시도 on Entry
- 브라우저 알림 권한 거부 on Room
- localStorage 삭제 후 재접속 on Entry
- NPC 스케일링 (0→5→10+ 실제 유저) on Room
- 빈 기록 상태 (첫 세션) on Result

## Critical Paths
- Entry → Room → Complete → Result (full happy path E2E)
- Entry → Room → Timeout → Result (timeout path E2E)
- Result → "다시 입장" → Entry → Room (re-entry loop E2E)
- Tab close → server-side expiry → re-open → Result (resilience E2E)
